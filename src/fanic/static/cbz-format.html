FANIC CBZ Interchange Profile 1.0

This white paper defines the CBZ profile used by FANIC for ingest, storage, and display. It uses common comic-reader practice (including GonVisor-style ComicInfo.xml metadata) as a baseline, then narrows behavior so producers and consumers get deterministic results.

1. Scope and Goals

This profile standardizes:

  • Archive structure and file naming conventions.
  • Metadata sources and precedence.
  • ComicInfo.xml field mapping into FANIC work metadata.
  • Page ordering rules.
  • Ingest-time normalization behavior.

It does not redefine ZIP itself and does not mandate reader UI behavior.

2. Archive Container

A FANIC CBZ is a ZIP archive with extension .cbz.

Supported image members for ingest discovery:

  • .jpg / .jpeg
  • .png
  • .webp
  • .gif
  • .avif

Directory structure inside the ZIP is allowed; all matching image entries are considered.

3. Metadata Sources and Precedence

FANIC reads metadata from the following sources, in order:

  1. In-app metadata form submission (highest priority).
  2. ComicInfo.xml in archive.
  3. Built-in defaults (lowest priority).

Notes: The load metadata action parses ComicInfo.xml for form prefill.

4. Page Ordering

Ordering rules are:

  1. Pages are sorted lexicographically by archive member path during ingest.
  2. Ordering can be adjusted after ingest using the editor page manager.

5. Image Normalization

During ingest, each page image is transcoded and stored as:

  • Page image format: AVIF
  • Page image quality: 60
  • Thumbnail format: WEBP (quality 82)

Any unreadable or non-decodable discovered image causes ingest failure.

6. ComicInfo.xml Mapping (GonVisor-Compatible Baseline)

FANIC treats ComicInfo.xml as the baseline metadata contract used by many comic readers including GonVisor. The following fields are mapped when present.

ComicInfo.xml Element FANIC Field Behavior
TitletitleTrimmed string.
SummarysummaryTrimmed string.
LanguageISOlanguageISO code, defaults to en.
Seriesseries, fandoms[0]Series string and initial fandom seed.
Numberseries_indexStored as provided, parsed downstream when needed.
Characterscharacters[]Comma-split, trimmed list.
Tagsfreeform_tags[]Comma-split, trimmed list.
AgeRatingratingNormalized to FANIC rating enum when alias is recognized.
Year + Month + Daypublished_atRendered as YYYY-MM-DD with missing month/day defaulted to 01.

7. Rating Enum

FANIC canonical ratings are:

  • Not Rated
  • General Audiences
  • Teen And Up Audiences
  • Mature
  • Explicit

Common aliases such as G, PG-13, M, and NC-17 are normalized during ingest.

8. Recommended Producer Layout

my-comic.cbz
|- ComicInfo.xml
|- 001-cover.jpg
|- 002-page.jpg
|- 003-page.jpg

Recommendations:

  • Use zero-padded page filenames.
  • Keep ComicInfo.xml at archive root.

9. Compliance Notes

  • CBZ without metadata is valid if it has at least one supported image.
  • ComicInfo.xml is optional but strongly recommended.
  • Unknown ComicInfo.xml elements are ignored.
  • This document defines FANIC behavior and may be versioned in future revisions.
