FANIC CBZ Ingest Profile (Current Behavior)

This document describes how FANIC currently ingests and normalizes CBZ uploads. It reflects actual runtime behavior in the site, including chapter import rules, page ordering, metadata parsing, and editor-side chapter management.

1. Scope and Goals

This profile standardizes:

  • Archive structure and file naming conventions.
  • Metadata sources and precedence.
  • ComicInfo.xml field mapping into FANIC work metadata.
  • Page ordering and optional explicit page_order override behavior.
  • Chapter auto-import from archive directories and chapter lifecycle rules.
  • Ingest-time normalization behavior.

It does not redefine ZIP itself and does not mandate reader UI behavior.

2. Archive Container

A FANIC CBZ is a ZIP archive with extension .cbz.

FANIC accepts most image formats that Pillow can decode, including:

  • .jpg / .jpeg
  • .png
  • .bmp
  • .webp
  • .gif
  • .avif
  • .tiff / .tif

Directory structure inside the ZIP is allowed; all matching image entries are considered.

3. Metadata Sources and Precedence

FANIC reads metadata from the following sources, in order:

  1. In-app metadata form submission (highest priority).
  2. ComicInfo.xml in archive.
  3. Built-in defaults (lowest priority).

Notes: The load metadata action parses ComicInfo.xml for form prefill.

4. Page Ordering

Ordering rules are:

  1. By default, pages are sorted by FANIC natural member sort (directory + filename with numeric awareness).
  2. If metadata includes page_order, FANIC prioritizes that explicit filename order first, then falls back to natural sort.
  3. Ordering can be adjusted after ingest using the editor page manager.

5. Chapters (Current FANIC Behavior)

FANIC chapter behavior is directory-driven at ingest time:

  1. Auto chapters are created only from non-root image directories in the CBZ (for example, chapter-01/, chapter-02/).
  2. Images stored at archive root are valid pages but do not create an automatic chapter.
  3. Auto chapter title is the directory path text (trimmed).
  4. Each chapter range (start_page/end_page) is derived from imported page positions after final ordering.
  5. Chapter page membership is stored explicitly, then kept in sync when pages are reordered, replaced, or deleted in the editor.
  6. If a chapter loses all members after edits, FANIC removes that chapter.

Re-importing pages from CBZ replaces previously imported auto chapters for that work, then rebuilds chapters from the current archive directory structure.

6. Image Normalization

During ingest, each page image is transcoded and stored as:

  • Page image format: AVIF
  • Page image quality: adaptive quality selection based on account page count policy
  • Thumbnail format: AVIF

Any unreadable or non-decodable discovered image causes ingest failure.

9. Rating Enum

FANIC canonical ratings are:

  • Not Rated
  • General Audiences
  • Teen And Up Audiences
  • Mature
  • Explicit

Ingest normalizes common aliases (such as G, PG-13, M, and NC-17), and export maps FANIC ratings to ComicInfo AgeRating values.

10. Recommended Producer Layout

my-comic.cbz
|- ComicInfo.xml
|- chapter-01/001-cover.jpg
|- chapter-01/002-page.jpg
|- chapter-02/003-page.jpg
|- extras/004-bonus.jpg

Recommendations:

  • Use zero-padded page filenames for stable natural sorting.
  • Use per-chapter directories if you want FANIC to auto-create chapters on import.
  • Keep ComicInfo.xml at archive root.

11. Compliance Notes

  • CBZ without metadata is valid if it has at least one supported image.
  • ComicInfo.xml is optional but strongly recommended.
  • Chapter import is optional; no directory structure means no auto-created chapters.
  • Unknown ComicInfo.xml elements are ignored during ingest.
  • FANIC-specific extensions are stored in schema-safe string fields (for example ScanInformation and Notes) for round-trip metadata recovery.
  • This document defines FANIC behavior and may be versioned in future revisions.
